F334 工程电流逻辑梳理

适用对象：原始 F334 Buck/Boost 工程 DmoeCBB02410。

目的：把 F334 工程中“电流量”从 ADC 采样、偏置处理、模式方向、显示、保护到调试输出的逻辑完整列出来，避免后续移植时只抄显示公式，却忽略整个电流语义体系。

本文档只基于当前已读取到的 F334 源码做循证整理；不臆造未在代码中出现的硬件参数。

1. F334 工程里有哪些“电流量”

在 function.h 的 struct _ADI 中，电流相关字段共有三组：

字段 含义 角色
Ix / IxAvg / IxOffsetX 侧电流、平均值、偏置X 侧通道
Iy / IyAvg / IyOffsetY 侧电流、平均值、偏置Y 侧通道
IL / ILAvg / ILOffset电感电流、平均值、偏置电感通道

这说明 F334 原工程从一开始就不是“只有一个输出电流变量”，而是同时维护了 X 侧、Y 侧、电感三套电流语义。

2. ADC 原始映射

在 ADCSample() 的注释和赋值中，可以确认 ADC1 五路采样语义是：

ADC1_RESULT[0] -> IL
ADC1_RESULT[1] -> Vy
ADC1_RESULT[2] -> Iy
ADC1_RESULT[3] -> Vx
ADC1_RESULT[4] -> Ix

即：

  • ADC1[4] = X 侧电流原始量
  • ADC1[2] = Y 侧电流原始量
  • ADC1[0] = 电感电流原始量

3. 偏置建立逻辑

F334 工程在 StateMInit() 中专门建立电流偏置：

  1. 系统上电进入 Init 状态；
  2. 连续累计 256 次电流通道 raw 值；
  3. 分别得到 IxOffset、IyOffset、ILOffset；
  4. 偏置就绪后，状态机转入 Wait。
IxOffset = average(ADC1_RESULT[4], 256次)
IyOffset = average(ADC1_RESULT[2], 256次)
ILOffset = average(ADC1_RESULT[0], 256次)

这说明 F334 原工程的电流处理不是“raw 直接当电流”，而是明确建立了去偏置前提。

4. 模式相关的电流方向逻辑

这是 F334 电流逻辑里最关键、也最容易在移植时丢失的一层。

在 ADCSample() 中，电流不是简单做 raw - offset，而是随 BUCK / BOOST 模式切换方向：

4.1 BUCK 模式

Ix = ADC1_RESULT[4] - IxOffset
Iy = ADC1_RESULT[2] - IyOffset
IL = ADC1_RESULT[0] - ILOffset

4.2 BOOST 模式

Ix = IxOffset - ADC1_RESULT[4]
Iy = IyOffset - ADC1_RESULT[2]
IL = ILOffset - ADC1_RESULT[0]

这说明 F334 原工程不是只做“扣零点”，而是已经把电流方向语义编码进软件了。也就是说，同一个 ADC 通道在不同拓扑下，正方向定义会翻转。

5. 负值裁剪逻辑

在 ADCSample() 中，去偏置并按模式翻转之后，F334 会把负值裁成 0：

if(SADC.Ix < 0) SADC.Ix = 0;
if(SADC.Iy < 0) SADC.Iy = 0;
if(SADC.IL < 0) SADC.IL = 0;

这意味着：

  • F334 软件逻辑最终使用的是单极性正电流语义；
  • 方向翻转发生在裁剪前；
  • 裁剪后各电流量默认只表达“有效正向电流”。

6. 平均处理逻辑

在 ADCSample() 中，电流再进入一阶滑动平均：

IxAvg = lowpass(Ix)
IyAvg = lowpass(Iy)
ILAvg = lowpass(IL)

具体实现形式是：

IxSum = IxSum + SADC.Ix - (IxSum >> 6)
IxAvg = IxSum >> 6

Iy 和 IL 同理。

也就是说，F334 使用时序上更平滑的 IxAvg / IyAvg / ILAvg，而不是每次都用瞬时采样值。

7. F334 中“输出电流”怎么定义

在显示与保护中，F334 并不是固定说“输出电流 = Iy”或“输出电流 = Ix”，而是按模式映射：

模式 输出侧 作为输出电流的变量
BUCKY 侧Iy / IyAvg
BOOSTX 侧Ix / IxAvg

这说明 F334 的“IOUT”本质上是一个模式语义量，不是某个固定 ADC 通道名。

8. F334 的显示公式

在 LCDshowVIout() 中：

8.1 BUCK 显示

itemp = SADC.IyAvg * 3300 >> 12

8.2 BOOST 显示

itemp = SADC.IxAvg * 3300 >> 12

然后用 LCDShowFnum() 以两位小数的形式显示。

因此 F334 的屏幕电流显示遵循：

显示电流 = 平均电流Q12值 × 33.00 / 4096

这是一条非常清晰的量纲链：Q12 满量程电流被定义为 33.00A。

9. F334 的调试打印公式

在 RealValuePrintf() 中，F334 明确写出了电流量程常量：

常量 值 含义
IX_MAX33.0X 侧电流满量程
IY_MAX33.0Y 侧电流满量程
IL_MAX33.0电感电流满量程

实际打印公式为：

IL_real = SADC.IL * 33.0 / 4096
IY_real = SADC.Iy * 33.0 / 4096
IX_real = SADC.Ix * 33.0 / 4096

这与 LCD 显示里的 *3300 >> 12 是同一体系，只是一个给浮点调试打印，一个给整数屏幕显示。

10. F334 的保护逻辑如何使用电流

10.1 输出过流保护 OCP

在 SwOCP() 中：

  • BUCK：用 SADC.Iy 作为输出电流
  • BOOST：用 SADC.Ix 作为输出电流

对应阈值：

  • MAX_BUCK_OCP_VAL = 1365，注释说明约等于 11 * 2^12 / 33
  • MAX_BOOST_OCP_VAL = 745，注释说明约等于 6 * 2^12 / 33

10.2 短路保护

在 ShortOff() 中：

  • BUCK：io = SADC.Iy，vo = SADC.Vy
  • BOOST：io = SADC.Ix，vo = SADC.Vx

判定条件是：

输出电流 > 阈值  且  输出电压 < 阈值

其中：

MAX_SHORT_I = 1862  ≈ 15 * 2^12 / 33

这再次证明 F334 的保护体系也是按 33A 满量程 Q12 量纲建立的。

11. F334 的控制目标里，电流怎么定义

在 GetVItarget() 中，电流目标 Ioref 也遵循同一量纲：

  • BUCK 最大目标电流：MAX_IREF1 = 1242，约等于 10A
  • BOOST 最大目标电流：MAX_IREF = 621，约等于 5A

LCD 模式下电位器到电流目标的换算：

BUCK:  iTemp = (SADC.IadjAvg * 3072 >> 12) - 200
BOOST: iTemp = (SADC.IadjAvg * 1843 >> 12) - 200

对应注释分别解释为 10A 和 5A 目标范围。

但当前读到的 CtlLoop.c 只直接实现了电压环，尚未在这次阅读片段中看到“电流环如何闭环调 duty”的完整源码入口。因此不能在本文档里臆造“电流一定参与了哪一级闭环控制”。

12. F334 电流逻辑总流程

ADC1_RESULT[4/2/0]
    ↓
StateMInit() 建立 IxOffset / IyOffset / ILOffset
    ↓
ADCSample() 按 BUCK/BOOST 模式决定符号方向
    ↓
raw 与 offset 做差 / 反向做差
    ↓
负值裁成 0
    ↓
得到 Ix / Iy / IL
    ↓
一阶低通得到 IxAvg / IyAvg / ILAvg
    ↓
按模式选择输出侧电流：BUCK->Iy, BOOST->Ix
    ↓
显示：*3300 >> 12
保护：与 33A 体系阈值比较
调试打印：*33.0 / 4096

13. 最关键的判定

  1. F334 原工程的电流逻辑是完整语义链，不是单一显示公式。
  2. 它明确包含：偏置建立、模式翻转、负值裁剪、平均处理、模式映射、显示量纲、保护量纲。
  3. 所以后续任何移植，如果只抄 *3300 >> 12 或只抄某个显示系数，而没有恢复“模式翻转 + 偏置 + 输出侧映射”，都会造成语义失真。
  4. 换句话说：F334 的电流逻辑不是“一个公式”，而是一整套带拓扑语义的处理体系。
